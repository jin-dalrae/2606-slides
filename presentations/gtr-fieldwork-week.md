## The test mixed too many questions and introduced avoidable noise

<p class="slide-ref">Testing Process · mistakes</p>

<div class="cards-2">
  <article><h3>Trust before value</h3><p>We requested sensitive documents before clearly showing what participants would receive.</p></article>
  <article><h3>Facilitator-dependent language</h3><p>Net impact, maturity, projection, and milestones required explanation.</p></article>
  <article><h3>Inconsistent sample values</h3><p>The report and dashboard used different illustrative emissions numbers.</p></article>
  <article><h3>Too much per session</h3><p>The full journey, Assessment 1/2, Report 1/2, dashboard, and sharing questions limited depth.</p></article>
</div>

<p class="slide-closing">Testing mistake: the sessions covered the whole journey, but did not isolate each research question.</p>

---

## Low fidelity and think-aloud exposed the right blockers

<p class="slide-ref">Testing Process · what worked</p>

<div class="media-split">
  <div>
    <div class="cards-2">
      <article><h3>Peer-test first</h3><p>Internal walkthroughs caught sequencing and script problems before stakeholder time.</p></article>
      <article><h3>Storyboard framing</h3><p>Every participant entered the same founder journey before comparing screens.</p></article>
      <article><h3>Think beyond task completion</h3><p>Participants surfaced trust, comprehension, sharing, and accessibility concerns.</p></article>
      <article><h3>Cheap A/B comparison</h3><p>Paper artifacts made structural alternatives easy to compare and revise.</p></article>
    </div>
  </div>
  <img src="assets/images/gtr/fieldwork/assessment-report-print.jpg" alt="Printed two-page instant impact assessment report used in sessions">
</div>

<p class="slide-closing">Testing Process conclusion: storyboard-first framing and low-fidelity A/B comparison worked; session scope and facilitator dependence did not.</p>

## Repeated blockers matter more than isolated enthusiasm

<p class="slide-ref">What we learned · about testing</p>

<div class="steps-3">
  <article><span class="steps-3__num">01</span><div><h3>Concept and usability interact</h3><p>If the benefit is unclear, a participant may reject the form before evaluating its interface.</p></div></article>
  <article><span class="steps-3__num">02</span><div><h3>A/B preference is directional</h3><p>Assessment 1/2 and Report 1/2 reactions suggest what to retest; they do not select a universal winner.</p></div></article>
  <article><span class="steps-3__num">03</span><div><h3>Evidence types should stay separate</h3><p>Founder, technical, investment, peer-design, and accessibility feedback answer different questions.</p></div></article>
</div>

---

## Each stakeholder lens exposed a different risk

<p class="slide-ref">What we learned · stakeholders</p>

| Participant lens | Strongest feedback | Decision |
| --- | --- | --- |
| Ted · founder | Value unclear; document upload felt risky | Show benefit first; manual inputs before files |
| Amali · data | Scope and dashboard terms unclear; wanted visual + explanation | Define terms; combine both report strengths |
| Caroline · investment | Preferred presets, Assessment 2, and Report 2 | Guided inputs and visual first-read |
| Josh · peer design | Preferred Assessment 1 reviewability and Report 2 readability | Preserve easy review inside guided flow |
| Brian · accessibility | Labels, hierarchy, and data requests created barriers | Accessible labels, clearer hierarchy, trusted inputs |

## The winning direction is a synthesis — not one untouched variant

<p class="slide-ref">What we learned · synthesis</p>

<div class="cards-2">
  <article><h3>Hybrid assessment</h3><p>Assessment 2 guidance + Assessment 1 visibility and easy answer review.</p></article>
  <article><h3>Layered report</h3><p>Report 2 visual overview + Report 1 explanation and methodology.</p></article>
  <article><h3>Storyboard-led sequence</h3><p>Discover → assess → instant report → dashboard remains the product journey.</p></article>
  <article><h3>Separate + / −</h3><p>Keep footprint and handprint distinct, with plain-language definitions before any derived net.</p></article>
</div>

<p class="slide-closing">What we learned conclusion: the next direction is a synthesis of the variants, not a single untouched winner.</p>

## Six challenges remain open

<p class="slide-ref">Changes &amp; Future Direction · challenges</p>

<div class="cards-3">
  <article><h3>Assessment navigation</h3><p>Guidance and easy review still need to work together.</p></article>
  <article><h3>Real workflow</h3><p>No real data, files, deployment, or calculation accuracy was tested.</p></article>
  <article><h3>Comprehension</h3><p>Core terms still depend on facilitator explanation.</p></article>
  <article><h3>Trust</h3><p>Privacy claims and trusted document handling remain untested.</p></article>
  <article><h3>Founder sample</h3><p>Only one participant was an actual startup founder.</p></article>
  <article><h3>Calculation accuracy</h3><p>The illustrative emissions model was not tested against real company data.</p></article>
</div>

## Feedback changes the prototype, not the storyboard sequence

<p class="slide-ref">Changes &amp; Future Direction · pivots</p>

| Before | After feedback |
| --- | --- |
| Storyboard journey | Keep: discover → assess → instant report → dashboard |
| Ask for documents before payoff | Show benefit first; manual fields before files |
| Assessment 1 or Assessment 2 | Hybrid: guided steps + visible scope + answer review |
| Report 1 or Report 2 | Layered: visual overview + explanation and methodology |
| Generic activity checklist | Business-model-specific questions and defaults |
| Horizontal dashboard cards | Vertical hierarchy with goals and milestones grouped |
| Public sharing default | Private by default; selective sharing for positive metrics |

---

## Next round: narrow, build, then test

<p class="slide-ref">Changes &amp; Future Direction · next steps</p>

<div class="steps-3">
  <article><span class="steps-3__num">01</span><div><h3>Build the hybrid</h3><p>Value proposition → guided assessment with review → visual report with expandable detail.</p></div></article>
  <article><span class="steps-3__num">02</span><div><h3>Separate the studies</h3><p>Test terminology, document trust, and assessment navigation as distinct questions.</p></div></article>
  <article><span class="steps-3__num">03</span><div><h3>Recruit more founders</h3><p>Then test real data and calculation credibility after the flow works without facilitation.</p></div></article>
</div>

<p class="slide-closing">Changes &amp; Future Direction conclusion: prove value, trust, independent comprehension, and the hybrid flow before expanding the dashboard.</p>
